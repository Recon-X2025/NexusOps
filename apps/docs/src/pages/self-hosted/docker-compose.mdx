# Docker Compose Deployment

Docker Compose is the recommended deployment method for single-server installations and teams up to ~100 concurrent users.

## Prerequisites

- Docker Engine 24.x or later
- Docker Compose v2.x (`docker compose` — note: no hyphen)
- A server with at least 4 GB RAM and 20 GB disk

## docker-compose.yml

Save the following as `docker-compose.yml`:

```yaml
version: "3.9"

services:
  postgres:
    image: postgres:15-alpine
    restart: unless-stopped
    environment:
      POSTGRES_USER: ${POSTGRES_USER:-nexusops}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
      POSTGRES_DB: ${POSTGRES_DB:-nexusops}
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER:-nexusops}"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    restart: unless-stopped
    command: redis-server --requirepass ${REDIS_PASSWORD}
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  api:
    image: ghcr.io/recon-x2025/nexusops-api:latest
    restart: unless-stopped
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
    environment:
      DATABASE_URL: postgresql://${POSTGRES_USER:-nexusops}:${POSTGRES_PASSWORD}@postgres:5432/${POSTGRES_DB:-nexusops}
      REDIS_URL: redis://:${REDIS_PASSWORD}@redis:6379
      APP_SECRET: ${APP_SECRET}
      JWT_SECRET: ${JWT_SECRET}
      NODE_ENV: production
      PORT: 3001
      ANTHROPIC_API_KEY: ${ANTHROPIC_API_KEY:-}
      SMTP_HOST: ${SMTP_HOST:-}
      SMTP_PORT: ${SMTP_PORT:-587}
      SMTP_USER: ${SMTP_USER:-}
      SMTP_PASSWORD: ${SMTP_PASSWORD:-}
      SMTP_FROM: ${SMTP_FROM:-noreply@nexusops.local}
    ports:
      - "3001:3001"

  web:
    image: ghcr.io/recon-x2025/nexusops-web:latest
    restart: unless-stopped
    depends_on:
      - api
    environment:
      NEXTAUTH_URL: ${NEXTAUTH_URL}
      NEXTAUTH_SECRET: ${NEXTAUTH_SECRET}
      NEXT_PUBLIC_API_URL: ${NEXT_PUBLIC_API_URL:-http://localhost:3001}
      NODE_ENV: production
    ports:
      - "3000:3000"

volumes:
  postgres_data:
  redis_data:
```

## Environment Variables

Create a `.env` file in the same directory as `docker-compose.yml`:

```env
# PostgreSQL
POSTGRES_USER=nexusops
POSTGRES_PASSWORD=changeme-use-strong-password
POSTGRES_DB=nexusops

# Redis
REDIS_PASSWORD=changeme-redis-password

# Application secrets (generate with: openssl rand -hex 32)
APP_SECRET=replace-with-64-char-random-string
JWT_SECRET=replace-with-64-char-random-string
NEXTAUTH_SECRET=replace-with-64-char-random-string

# Web app
NEXTAUTH_URL=https://nexusops.yourcompany.com
NEXT_PUBLIC_API_URL=https://nexusops.yourcompany.com/api

# AI (optional)
ANTHROPIC_API_KEY=sk-ant-...

# Email (optional)
SMTP_HOST=smtp.mailgun.org
SMTP_PORT=587
SMTP_USER=postmaster@mg.yourcompany.com
SMTP_PASSWORD=your-smtp-password
SMTP_FROM=nexusops@yourcompany.com
```

> **Security**: Never commit `.env` to version control. Restrict file permissions: `chmod 600 .env`.

## Starting NexusOps

```bash
# Start all services in the background
docker compose up -d

# Check service status
docker compose ps

# View logs for all services
docker compose logs -f

# View logs for a specific service
docker compose logs -f api
```

## Running Migrations

After the first start (or after upgrading), run database migrations:

```bash
docker compose exec api pnpm --filter @nexusops/db db:migrate
```

## Stopping and Restarting

```bash
# Stop all services (data is preserved in volumes)
docker compose down

# Stop and remove all data volumes (DESTRUCTIVE — deletes all data)
docker compose down -v

# Restart a single service
docker compose restart api
```

## Upgrading

```bash
# Pull the latest images
docker compose pull

# Recreate containers with new images
docker compose up -d

# Run migrations (always after upgrade)
docker compose exec api pnpm --filter @nexusops/db db:migrate
```

Pin to a specific version by replacing `latest` with a version tag (e.g., `v1.2.3`) in `docker-compose.yml`. This prevents unintended upgrades.

## Monitoring

Check the API health endpoint to verify the service is running:

```bash
curl http://localhost:3001/api/health
# Expected: {"status":"ok","version":"1.x.x","db":"connected","redis":"connected"}
```

## Troubleshooting

**API fails to start with "connection refused" to postgres:**
Ensure `postgres` container is healthy before `api` starts. Check with `docker compose ps` — `postgres` should show `healthy`.

**"Prisma migration failed" during first start:**
Verify `DATABASE_URL` in `.env` matches the `POSTGRES_*` variables. The user must have `CREATE TABLE` privileges.

**Web shows blank screen after deploy:**
Check `NEXTAUTH_URL` — it must exactly match the URL you're accessing in the browser (including `https://` and no trailing slash).
